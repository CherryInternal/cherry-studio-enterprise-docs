## エディション概要

Cherry Studio エンタープライズ版には 2 つのエディションがあります。ニーズに合わせてお選びください：

| | **Express 版**（クラウドホスト型） | **Premium 版**（プライベート導入型） |
| --- | --- | --- |
| **導入方式** | 導入不要、登録するだけですぐ利用可能 | Docker / Helm で自社導入 |
| **データ保存** | Cherry Studio クラウドで管理 | お客様のサーバーに完全保存 |
| **適した用途** | 素早く開始したい中小チーム | データコンプライアンス重視の大規模組織 |
| **運用** | Cherry Studio が保守 | お客様自身で運用 |
| **課金方式** | サブスクリプション（Alipay／銀行振込） | ライセンスベース |

</Callout>

## データセキュリティと隔離

どちらのエディションを選んでも、Cherry Studio エンタープライズ版はデータ保護を多層的に提供します：

| セキュリティ | Express 版 | Premium 版 |
| --- | --- | --- |
| **データ隔離** | グループベースの厳格な隔離——各組織のデータは完全に分離され、他の組織からは参照不可 | 独立導入による物理的隔離 |
| **通信暗号化** | 全通信 HTTPS 暗号化 | お客様側で SSL/TLS を設定 |
| **認証** | JWT + OAuth2 認証、パスワードは暗号化保存 | JWT 認証、パスワードは暗号化保存 |
| **アクセス制御** | ロールベースのきめ細かな権限体系（管理者 / グループ管理者 / メンバー） | 同左 |
| **API セキュリティ** | 独立した API Key、ユーザーごとのアクセス権隔離 | 同左 |

<Callout type="info" title="Express 版のデータ隔離メカニズム">
Express 版は共有インフラ＋アプリケーション層隔離アーキテクチャを採用しています。各組織のデータ（モデル設定、知識ベース、アシスタント、ファイルなど）はグループ隔離メカニズムにより厳密に分離されます。すべての API リクエストは認証と権限チェックを経由し、自組織のデータのみアクセス可能です。
</Callout>
