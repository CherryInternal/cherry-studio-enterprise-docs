<Callout type="info" title="バージョン要件">
  ワンクリック更新には **v0.3.3** 以降が必要です。現在のバージョンが v0.3.3 未満の場合は、まず手動で
  v0.3.3 に更新してからワンクリック更新を利用してください。
</Callout>

本ドキュメントでは Cherry Studio Enterprise を最新バージョンへ更新する方法を説明します。管理コンソールのワンクリック更新（Docker Compose 導入）または手動更新に対応します。

## 更新方法

### 方法 1：管理コンソールのワンクリック更新（推奨）

Docker Compose で導入している場合、管理コンソールから直接更新できます。

### 前提条件

ワンクリック更新を有効にするため、`docker-compose.yml` に以下を設定します：

1. **Docker Socket をマウント**：コンテナ内から Docker を操作
2. **docker-compose.yml をマウント**：設定ファイルの読み取り/更新を許可

### 設定例

```yaml
services:
  api:
    image: cherrystudio/cherry-studio-enterprise-api:v0.3.2
    container_name: cherry-studio-enterprise-api
    ports:
      - '3670:3670'
      - '3680:3680'
    environment:
      # ... 他の環境変数 ...
    volumes:
      # アプリデータ
      - ./data:/app/data

      # ワンクリック更新に必須のマウント
      - /var/run/docker.sock:/var/run/docker.sock # Docker Socket をマウント
      - ./docker-compose.yml:/app/docker-compose.yml # docker-compose.yml をマウント
    restart: unless-stopped
```

### 手順

1. 管理コンソールにログイン
2. **About** ページへ移動
3. **バージョン情報** セクションの **更新を確認** をクリック
4. 新バージョンがあれば **今すぐ更新** ボタンとバージョン一覧が表示
5. 更新するバージョンを選択し **更新** をクリック
6. システムが自動で以下を実行：
   - 新しいイメージを取得
   - docker-compose.yml のイメージタグを更新
   - 旧コンテナを停止
   - 新コンテナを起動
7. 更新完了後、ページが自動でリフレッシュ

### 注意事項

- 更新中はサービスが短時間停止します（通常 10〜30 秒）
- 更新中にブラウザを閉じたり更新しないでください
- 更新に失敗した場合はコンテナログを確認

### 方法 2：手動更新

ワンクリック更新を使わない場合は以下の手順です：

### 1. 新しいイメージを取得

```bash
# 最新版を取得
docker pull cherrystudio/cherry-studio-enterprise-api:latest

# 特定バージョンを取得
docker pull cherrystudio/cherry-studio-enterprise-api:v0.3.2
```

### 2. docker-compose.yml を更新

`docker-compose.yml` のイメージタグを変更します：

```yaml
services:
  api:
    image: cherrystudio/cherry-studio-enterprise-api:v0.3.2 # バージョン更新
    # ... 他の設定はそのまま ...
```

### 3. サービス再起動

```bash
# 停止して再作成
docker-compose down
docker-compose up -d

# ログで起動確認
docker-compose logs -f
```

### 4. 更新確認

- 管理コンソールの **About** でバージョン確認
- サービス正常性を確認：`curl http://localhost:3670/health`

## バージョン情報の確認

### 現在のバージョン

- **管理コンソール**：ログインして **About** を開く
- **API**：`/health` エンドポイントへアクセス
- **CLI**：
  ```bash
  docker inspect cherry-studio-enterprise-api | grep "Image"
  ```

### 更新ログの確認

- **管理コンソール**：**About** > **更新ログ**
- **オンライン文書**：[CHANGELOG](https://releases.enterprise.cherry-ai.com/api/changelog)

## よくある質問

### 1. ワンクリック更新ボタンが表示されない

**原因の可能性**：

- Docker Socket がマウントされていない
- docker-compose.yml がマウントされていない
- Docker Socket の権限不足

**対処方法**：

- `docker-compose.yml` に必要な volume があるか確認
- Docker Socket の存在を確認：`ls -la /var/run/docker.sock`
- コンテナが Docker Socket にアクセスできるか確認

### 2. イメージ取得に失敗

**原因の可能性**：

- ネットワーク接続の問題
- Docker Hub へのアクセス制限

**対処方法**：

- ネットワークを確認
- Docker レジストリミラーを設定
- 手動でイメージ取得後に更新

### 3. 更新後にサービスが起動しない

**確認手順**：

```bash
# コンテナログを確認
docker logs cherry-studio-enterprise-api

# コンテナ状態を確認
docker ps -a

# ポート競合を確認
netstat -tulpn | grep -E '3670|3680'
```

**よくある原因**：

- DB 接続設定の誤り
- ポート競合
- 環境変数の不足
