# 3. FAQ

## ❓ よくある質問とベストプラクティス

### 「デフォルトグループ」について

システム初期化時に **デフォルトグループ** が自動作成されます。

* **自動割り当て:** SSO、招待リンク、または一括インポートで登録された新規ユーザーは、グループ指定がない場合デフォルトに割り当てられます。
* **ベストプラクティス:**

    <div data-gb-custom-block data-tag="hint" data-style="info" class="hint hint-info"><p>デフォルトグループの権限は <strong>最小限</strong>（例：無料/低コストモデルと基本アシスタントのみ）に設定することを推奨します。正式な部署に割り当てられる前に高額な API コストが発生するのを防げます。</p></div>

### トラブルシューティング

### Q: グループでモデルを選択したのにクライアントで見えません。

**A:** 次を確認してください：

1. **グローバルスイッチ:** **モデル管理** で対象モデルが **有効** になっているか確認。全体で無効化されていると利用できません。
2. **同期遅延:** クライアントの設定が同期されているか確認。**設定 > 更新チェック**、または再起動を案内してください。

### Q: ユーザーは複数グループに所属できますか？

**A:** **できません。** Cherry Studio では 1 ユーザーは同時に 1 グループのみです。複合権限が必要な場合は、統合権限を持つ新しいグループ（例：「開発部長グループ」）を作成してください。

### Q: 特定モデルが検索に出てきません。

**A:** グループ管理のモデル一覧は **モデル管理** から取得します。モデルが表示されない場合、そのモデルが未登録です。**モデル管理** > **モデル追加** で Model ID を追加し保存してください。
