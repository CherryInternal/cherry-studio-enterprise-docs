# 2. リソース権限設定

### 1. アシスタント権限の設定

Cherry Studio クライアント左側の「アシスタント」欄に表示されるプリセットエージェントを制御します。

* **すべてのアシスタント**：選択すると企業ライブラリの全アシスタントにアクセス可能。
* **指定アシスタント**：**（推奨）** 必要なアシスタントのみを選択し、UI を簡潔に保つ。

**手順：**

1. 作成/編集画面の **利用可能なアシスタント** を探す。
2. **指定アシスタント** を選択。
3. ドロップダウンで対象アシスタントを選択（例：`翻訳アシスタント`、`コードレビュー担当`）。

![](/assets/images/zh/admin/group-management-resource-permissions-1_zh.png)

### 2. モデル権限の設定

チャット画面上部のモデル選択で使用できる LLM を制御します。

<Callout type="warn">
**コスト制御のヒント:** モデル権限の設定は API 支出を抑える最も有効な手段です。GPT-4 など高価なモデルはコア部門に限定することを推奨します。
</Callout>

**手順：**

1. **利用可能なモデル** を見つける。
2. **指定モデル** を選択。
3. 設定済みの **Model Provider** が一覧表示されます。
4. 以下が可能：
   * **プロバイダー単位で全選択：** `OpenAI` や `DeepSeek` 横のチェックで一括許可。
   * **モデル単位で選択：** プロバイダーを展開し Model ID（例：`gpt-5`、`deepseek-coder`）を指定。
5. 検索ボックスでモデル名を検索。
6. **選択済みのみ表示** にチェックして割り当て内容を確認。

![モデルセレクター UI](/assets/images/zh/admin/group-management-model-selector-ui_zh.png)

### 3. 権限設定の検証

正しく設定されているか確認するため、簡易テストを行います：

1. テストユーザーを作成し新グループに割り当て。
2. テストアカウントでクライアントにログイン。
3. **チェック A:** 新規チャットでモデル一覧を開き、選択したモデル **のみ** が表示されることを確認。
4. **チェック B:** アシスタントライブラリで許可済みのアシスタントのみ表示されることを確認。
