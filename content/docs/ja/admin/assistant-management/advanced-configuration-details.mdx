作成後に **編集** をクリックすると詳細設定画面に入ります。左ナビで 4 つの主要領域に分かれています。

## 3.1 プロンプト設定 (Prompt Settings)

アシスタントの「人格」と権限を管理します。

<img src="/assets/images/ja/admin/assistant-management-advanced-config-1_ja.webp" alt="" />
* **プロンプトエディタ:** Markdown 対応。システム指令を随時更新。
* **プロンプト可視性:**

  * **On:** ユーザーが対話詳細でシステムプロンプトを確認可能。
  * **Off:** 管理者のみ閲覧可。企業のコア Prompt 資産を保護。

  <img src="/assets/images/ja/admin/assistant-management-advanced-config-2_ja.webp" alt="" />

## 3.3 知識ベース設定 (Knowledge Base)

アシスタントに「外部知識」を付与します。**[知識ベース管理]** で作成済みのデータセットを選択できます。

* **動作:** 有効化すると、回答前に選択した知識ベースを検索し、関連するコンテンツをコンテキストとしてモデルに提供します。

<img src="/assets/images/ja/admin/assistant-management-advanced-config-5_ja.webp" alt="" />
## 3.4 MCP サーバー (MCP Servers)

このモジュールはアシスタントの「ツールボックス」です。MCP (Model Context Protocol) サーバーを有効化することで、Web 検索、チャート生成、ファイル IO などの実行能力を持たせ、単なる対話者から実行者へ進化させます。

<img src="/assets/images/ja/admin/assistant-management-advanced-config-6_ja.webp" alt="" />
このページには **グローバル設定** で追加済みの MCP サービスが表示されます。スイッチで付与できます。

**ツール例:**
* **Playwright:** アシスタントがインターネットにアクセスし、Web ページの内容を読み取ることを許可します。
* **Filesystem:** アシスタントがサーバー上の特定のファイルディレクトリを読み書きすることを許可します。

| 項目 | 説明 |
| -------------------- | ---------------------------------------------- |
| **名称 (Name)** | MCP サービスの一意 ID（例：`fetch`、`mcp-server-chart`）。 |
| **タイプ (Type)** | 接続方式（通常 `stdio` または `sse`）。 |
| **説明 (Description)** | サービスの概要。 |
| **有効 (Enable)** | **[中核操作]** スイッチ。ON でツール呼び出しが可能。 |

**代表的な利用シーン**

* **Web アクセス (`fetch` / `search`)**：ニュース要約やデータ取得。
* **可視化 (`mcp-server-chart`)**：CSV などのデータからグラフ生成。
* **ファイル操作 (`filesystem`)**：ログやコードの読み取り分析。

<Callout type="info">
**前提条件:** これらのサービスは **MCP サーバー（グローバル）** で事前に設定されている必要があります。空の場合は管理者に依頼してください。
</Callout>

<Callout type="info">
**設定の推奨:** 複数 MCP を有効化できますが、**「最小限」** を推奨します。

* 例：「データ分析アシスタント」には `mcp-server-chart` と `filesystem` のみ。
* 理由：ツールが多すぎると注意が分散し、ツール呼び出し失敗や幻覚が増える可能性があります。
</Callout>
