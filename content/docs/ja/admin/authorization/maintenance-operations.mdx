# 3. 保守操作

ページ下部でライセンスの更新や移行操作ができます。

### ライセンス情報の更新

更新や容量拡張（ユーザー数追加など）後、ローカルに反映されない場合があります。

1. 下部の **情報を更新** ボタンをクリック
2. ライセンスサーバーから最新情報を取得
3. **成功の目安:** 「有効期限」や「ユーザー上限」が最新値に更新される

### License の撤回（ライセンス移行）

サーバーのハードウェア更新や新しい私有クラウドへの移行時は、現在のライセンスを解放してから新しい環境で有効化してください。

1. **情報を更新** の横にある **┇（その他）** をクリック
2. メニューから **License を撤回** を選択
3. 必要に応じて確認文字列を入力

<Callout type="error">
**危険操作の警告：**

**License を撤回** すると、現在のサーバーとの紐付けが即時解除されます。

* 直ちに「未ライセンス」状態になります。
* 管理者以外のユーザーはログインできなくなる可能性があります。
* **新しい License を用意している場合、またはこのサーバーを停止する場合以外は実行しないでください。**
</Callout>

***

### FAQ

**Q: 情報更新が失敗する理由は？**

A: サーバーのネットワーク接続を確認してください。完全な社内ネットワーク（インターネット非接続）では自動更新が失敗します。サポートに連絡して新しいオフライン License を入手し、再度インポートしてください（未ライセンス状態でインポート入口が表示されます）。

**Q: 「ユーザー上限」に達したら？**

A: 次の 2 つの方法があります：

1. **ユーザー管理** で退職者アカウントを削除/無効化し枠を確保
2. Cherry Studio の営業担当に追加ユーザー枠（User Seat）を購入

**Q: License が期限切れになるとデータは消えますか？**

A: **消えません。** 期限切れはサービス利用制限のみ（対話不可、ログイン不可など）で、知識ベースやモデル設定、会話履歴は DB に残ります。新しい有効な License を導入すれば即座に復旧します。
