# 4. トラブルシューティング

API Key でモデル呼び出しが失敗する場合、主に以下の原因に分類できます。エラーコード/メッセージに沿って確認してください。

### Q1: "Insufficient Balance" または "Quota Exceeded"（残高不足）

**症状:** `402 Payment Required` または `429 Too Many Requests` が返る。

* **原因:**
  1. **上流の残高不足:** 企業が紐付けたプロバイダー（OpenAI、DeepSeek 公式など）の残高切れ。
  2. **ゲートウェイ制限:** Cherry Studio のレート制限が一時的に発動。
* **対処:**
  * **モデル管理** で API Key が有効か確認。
  * 上流プロバイダー（例：OpenAI Platform）で残高を確認。
  * **一時対応:** **グループ管理** でデフォルトモデルを残高のあるチャネルに切替。

### Q2: "Model Not Found"（モデルが存在しない）

**症状:** `404 Not Found` と `The model 'xxx' does not exist` が返る。

* **原因:**
  * リクエストのモデル名（例：`gpt-4-turbo`）がグループの許可リストにない。
* **対処:**
  1. **リクエスト確認:** ユーザーコードの `model` パラメータを確認。
  2. **グループ権限確認:** **グループ管理** > 対象グループ > **モデルルーティング** でモデルを選択済みか確認。
  3. **エイリアス確認:** `gpt-4` を `gpt-4-0613` にマップしていないか確認。

### Q3: "401 Unauthorized"（認証失敗）

**症状:** API が接続を拒否。

* **原因:**
  * Key の入力ミス（余分な空白など）。
  * Key が無効化/期限切れ。
* **対処:**
  * **キー設定** で **有効状態** を確認。
  * **有効期限** を確認。
  * 必要に応じて古い Key を削除し、新しい `sk-` Key を発行。

### Q4: "No Healthy Upstream"（有効な上流なし）

**症状:** `502 Bad Gateway` または `503 Service Unavailable` が返る。

* **原因:**
  * グループに紐付いたモデルチャネルが全てオフラインまたはタイムアウト。
* **対処:**
  * **モデル管理** で **Check** をクリックして接続確認。
  * サーバーから外部 API（例：api.openai.com）へ到達できるか確認。
