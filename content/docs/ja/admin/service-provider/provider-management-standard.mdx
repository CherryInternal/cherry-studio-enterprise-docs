# 1. プロバイダー管理（Standard/BYOK）

### 概要

**パス：** 管理コンソールサイドバー > **プロバイダー管理**

<img src="/assets/images/zh/admin/provider-management-standard-1_zh.webp" alt="" />
このモジュールでは、企業が利用する AI モデルのソースを統合的に管理できます。Cherry Studio は主要プロバイダー用のプリセットを備えており、OpenAI、Anthropic（Claude）、Google（Gemini）に加え、OpenAI 互換の国内中継プロバイダー（SiliconFlow、DeepSeek、OpenRouter など）にも対応します。

<Callout type="info">
**Standard 版の範囲**：標準の API Key と API Host で接続する商用モデルサービスを指します。
</Callout>

***

### 1. 新規プロバイダー追加

**操作：** 一覧下部の **+ プロバイダー追加** をクリック。

<img src="/assets/images/zh/admin/provider-management-standard-2_zh.webp" alt="" />
プリセットテンプレートで主要プロバイダーを素早く接続できます。

### 3. モデルパラメータと課金設定

**操作：** モデル右側の **⚙️ 設定** アイコンをクリック。

企業の運用コストを正確に計算するため、モデルごとの価格を設定してください。

<img src="/assets/images/zh/admin/provider-management-standard-4_zh.webp" alt="" />
#### モデル設定の編集

* **モデル ID**：API 呼び出しに使用される文字列（変更不可、公式ドキュメントと一致必須）。
* **コンテキスト長**：最大メモリウィンドウ（例：`32k`, `128k`）。
* **入力/出力価格**：
  * **入力価格**：100 万トークンあたりの入力料金。
  * **出力価格**：100 万トークンあたりの出力料金。
  * **通貨**：`CNY` または `USD`。

<Callout type="warn">
**価格設定が重要な理由**：ここでの単価は **用量統計** に反映されます。空欄だとコスト計算ができず、財務レポートが欠落します。
</Callout>

***

### 4. 接続確認

API Key と接続性の検証を行います：

<img src="/assets/images/zh/admin/provider-management-standard-5_zh.webp" alt="" />
1. プロバイダー右側の **接続確認** をクリック。
2. ダイアログでモデルを選択（例：`deepseek-chat`）。
3. **確認** をクリック。
4. **成功表示**：右下に「チェック成功、API Key 有効」。

### FAQ

**Q: プロバイダーを追加したがモデル一覧が空です。**

A: OpenAI 互換の一部プロバイダーは `/v1/models` に未対応です。下部の **+ モデル追加** で Model ID（例：`qwen-max`）を手動入力してください。

**Q: アシスタントが知識ベースを使えません。**

A: 使用中のモデルに 🔧（ツール）アイコンがあるか確認してください。軽量モデルや旧モデルはツール呼び出し非対応のため、知識ベース検索が実行できません。
