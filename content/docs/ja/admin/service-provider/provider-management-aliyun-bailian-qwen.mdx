## 概要

**パス：** 管理コンソールサイドバー > **プロバイダー管理**

Cherry Studio Enterprise は既定で **阿里云百炼 (Aliyun Bailian)** を中核エンジンとして統合しています。OpenAI 互換プロトコルがプリセットされているため、API Key を入力するだけで Qwen 全系列と百炼ホストの高性能 OSS モデルを利用できます。

<Callout type="info">
**既定戦略と拡張性**

* **既定状態**：初期状態では「阿里云百炼」のみが表示されます。
* **拡張**：管理者が他のプロバイダー（例：ローカル Ollama）を追加し、複数チャネルを統合できます。
</Callout>

***

## 1. 既定プロバイダーを有効化

一覧には **"阿里云百炼"** カードが表示されています。

<img src="/assets/images/ja/admin/provider-management-aliyun-bailian-qwen-1_ja.webp" alt="" />
1. **阿里云百炼** カードをクリックし、右上のトグルをクリックしてプロバイダーを有効化。
2. **API Key**：阿里云百炼のコンソールで取得したキーを入力（`sk-` で始まる）。
3. **API Host**：既定値のまま。
   * _アドレス_: `https://dashscope.aliyuncs.com/compatible-mode/v1`
4. **接続確認** をクリックし「API Key 有効」を確認。
5. **保存**。

***

## 3. モデル一覧の管理

<img src="/assets/images/ja/admin/provider-management-aliyun-bailian-qwen-2_ja.webp" alt="" />
モデルが多いため、従業員が迷わないよう整理を推奨します。

1. 百炼カード下の **𝌡 モデル管理** をクリック。
2. **推奨フィルタ**：
   * `qwen3-max` と `qwen3.6-plus`（最新世代の商用モデル）を残す。
   * `qwq-plus`（推論向け）を残す。
   * `qwen3-coder-plus`（開発向け）を残す。
   * `qwen3.5-flash`（軽量・高頻度タスク）を残す。
   * 必要に応じて `qwen3-32b` や `qwen3-8b`（OSS 代替）を残す。
3. **冗長モデルの非表示**：旧版モデル（`qwen-max`、`qwen-plus` など）、古い日付、小パラメータ（`0.5b`、`1.7b`）を **-** で非表示。
4. **保存**。

## 4. 他プロバイダーの追加（カスタム）

百炼が既定ですが、Cherry Studio は他チャネルも統合できます。

**例：**

* **OpenAI** の `gpt-4o` を使いたい。
* ローカルで **Ollama** を運用している。

**手順：**

1. 一覧下部の **+ プロバイダー追加** をクリック。
2. **プロバイダー種別**：
   * **OpenAI**（汎用互換）
   * **Anthropic**（Claude 系）
   * **Ollama**（ローカル）
3. **API Key** と **API Host** を入力。
   * _例: ローカル Ollama Host_: `http://192.168.1.5:11434/v1`
4. 追加したプロバイダーのモデルは、百炼モデルと同じ一覧に表示されます。

## 5. FAQ

**Q: `qwen3-max` と旧版 `qwen-max` の違いは？**

A: `qwen3-max` は最新世代の旗艦モデル（262K コンテキスト）、`qwen-max` は前世代（Qwen 2.5-Max）を指します。旧版はまだ利用可能ですが、新規プロジェクトには `qwen3-max` を推奨します。

**Q: `qwq-32b` と `qwq-plus` の違いは？**

A: `qwq-32b` は OSS 重みの推論モデル、`qwq-plus` は阿里云の商用クローズド推論モデルです。後者の方が性能/安全性が高い一方、コストも高めです。

**Q: `qwen-turbo` はまだ使えますか？**

A: `qwen-turbo` は更新停止済みです。`qwen3.5-flash` への移行を推奨します。コストがより低く、コンテキストも長い（1M）です。
