## ログ構成

バージョンは新しい順に表示され、各カードに以下が含まれます：

* **バージョン:** 例 `v0.4.10`。
* **公開日:** 例 `2026/03/30`。
* **変更点:** 具体的な機能説明。

## 変更種別の凡例

更新内容は色分けラベルで分類されます：

<table><thead><tr><th width="193.39453125">種別</th><th width="73.84375">色</th><th width="298.45703125">意味</th><th>例</th></tr></thead><tbody><tr><td><strong>新機能</strong> (New Feature)</td><td>緑</td><td>最適化ではなく新しい能力追加。</td><td><code>ユーザーアカウントの有効化/無効化に対応</code></td></tr><tr><td><strong>修正</strong> (Fix)</td><td>橙</td><td>既知バグや不具合の修正。</td><td><code>Ollama プロバイダーのメタデータ異常と埋め込みモデルルーティングの問題を修正</code></td></tr><tr><td><strong>改善</strong> (Improvement)</td><td>青</td><td>性能改善、UI 調整、体験向上。</td><td><code>Anthropic リクエストに Authorization Header を追加</code></td></tr></tbody></table>

<Callout type="info">
**ベストプラクティス:** 異常（統計不正確、レイアウト崩れ等）が発生した場合、最新の **修正** 項目に既知問題がないか確認してください。
</Callout>
