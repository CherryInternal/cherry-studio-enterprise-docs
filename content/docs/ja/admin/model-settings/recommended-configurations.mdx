# 4. おすすめ構成

意思決定を支援するため、企業タイプ別の推奨構成を提示します：

### 🚀 プラン A：性能優先（R&D/アルゴリズムチーム）

* **既定アシスタント**：`DeepSeek-Reasoner`（深い推論で複雑なコード問題に対応）
* **トピック命名**：`Qwen-Turbo`（高速タイトル生成）
* **翻訳モデル**：`DeepSeek-V3.2`

### ⚖️ プラン B：コストバランス（一般事務/管理）

* **既定アシスタント**：`DeepSeek-V3.2`（速度と知能のバランス、R1 の数分の一コスト）
* **トピック命名**：`Qwen-2.5-7B`（低コストのローカル/クラウドモデル）
* **翻訳モデル**：`DeepSeek-V3.2`

### 💰 プラン C：超高速応答（カスタマーサポート/営業）

* **既定アシスタント**：`GPT-5.1-mini` または `DeepSeek-Lite`（即時応答）
* **トピック命名**：`GPT-5.1-mini`
* **翻訳モデル**：`GPT-5.1-mini`

***

### FAQ

**Q: ここで使用中のモデルを削除するとどうなる？** A: **モデル管理** で既定モデルを削除すると、内蔵初期設定に戻るかエラーになる可能性があります。削除前に別モデルへ切り替えてください。

**Q: トピック命名モデルが弱いとどうなる？** A: 指示追従が弱いモデル（初期の 1B など）だと、"タイトルに要約"の文自体をタイトルにしたり、文字化けのようなタイトルになることがあり、履歴一覧が読みにくくなります。
