# サービス導入

Cherry Studio エンタープライズ版は複数の導入方式に対応しています。インフラに合わせて選択してください：

## 導入方式の比較

| 方式 | 難易度 | 利用シーン | 詳細ドキュメント |
|------|------|----------|----------|
| クラウドアプリマーケット | ⭐ 簡単 | 迅速な体験、小規模チーム | [表示](/ja/setup/cloud-app) |
| Docker Compose | ⭐⭐ 中程度 | 単一ノード導入、開発/テスト | [表示](/ja/setup/docker) |
| Kubernetes (Helm) | ⭐⭐⭐ 難しい | 本番環境、高可用性 | [表示](/ja/setup/helm) |

## クイック導入 (Docker)

すぐに試したい場合は Docker Compose をおすすめします：

```bash
# 設定をクローン
git clone https://github.com/CherryHQ/cherry-studio-enterprise
cd cherry-studio-enterprise

# サービス起動
docker compose up -d
```

起動後のアクセス先：
- 管理コンソール：`http://localhost:3680`
- API サービス：`http://localhost:3670`

## 次のステップ

導入完了後、[管理コンソール設定](/ja/quickstart/admin-setup) に進んでください。
