# Deploy the Service

Cherry Studio Enterprise supports multiple deployment options. Choose based on your infrastructure:

## Deployment Options

| Method | Difficulty | Use Case | Documentation |
|------|------|----------|----------|
| Cloud App Marketplace | ⭐ Easy | Quick trials, small teams | [View](/en/setup/cloud-app) |
| Docker Compose | ⭐⭐ Medium | Single-node deployment, dev/testing | [View](/en/setup/docker) |
| Kubernetes (Helm) | ⭐⭐⭐ Advanced | Production, high availability | [View](/en/setup/helm) |

## Quick Deployment (Docker)

If you just want a quick trial, we recommend Docker Compose:

```bash
# Clone config
git clone https://github.com/CherryHQ/cherry-studio-enterprise
cd cherry-studio-enterprise

# Start services
docker compose up -d
```

After the services start:
- Admin console: `http://localhost:3680`
- API service: `http://localhost:3670`
