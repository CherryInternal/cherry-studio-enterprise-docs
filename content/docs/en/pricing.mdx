Cherry Studio offers flexible pricing for users and enterprises of all sizes, from individual developers to large organizations.

## Version Selection Guide

### Express vs Enterprise: Key Differences

**Same functionality**: Express and Enterprise are identical in features and both include the full enterprise capability set.

**Only two core differences**:

1. **Provider configuration**

   - Express: built-in providers only, out of the box, no configuration required
   - Enterprise: fully configurable, add custom providers and private models

2. **Pricing model**
   - Express: from ¥5,000, no user limit
   - Enterprise: from ¥50,000 (1-50 users), tiered by headcount

### How to choose?

- **Choose Express**: want fast deployment, use mainstream LLM services, no need for custom providers
- **Choose Enterprise**: need private models, self-hosted services, or special provider requirements

## Detailed Plan Descriptions

### Community

---

**Target users**: Individuals, developers
**Deployment**: Single-machine use
**Technical support**: Community support
**License**: AGPL 3.0 open-source license

**Core features**:

- Completely free, no time limit
- Basic chat features for mainstream AI models
- Local deployment with full data control
- Built-in 30+ providers (OpenAI, Anthropic, Google Gemini, domestic LLMs, etc.)
- **Feature-rich but manual configuration**: users must apply for and configure API keys

**Provider usage**:

- Full provider list available, but users must register accounts themselves
- Manual configuration of API Key, Base URL, and other settings
- Best for technical users who are comfortable managing API keys

**Use cases**:

- Personal AI assistant
- Developer learning and trials
- Internal tools for small teams
- Cost-sensitive individual users

### Express

---

**Target users**: Startups, small and mid-sized teams
**Deployment**: Private deployment
**Technical support**: Dedicated technical support (annual service fee required)
**Pricing**: From ¥5,000, no user limit

**Core features** (same as Enterprise):

- ✅ Core features: complete AI chat and multi-model support
- ✅ Provider management: built-in mainstream LLM services (no customization)
- ✅ Model management: manage and configure available AI models
- ✅ Employee management: enterprise user accounts and org structure
- ✅ Shared knowledge base: enterprise vector knowledge base and document management
- ✅ Access control: role-based access control (RBAC)
- ✅ Data backup: full backup and recovery mechanisms
- ✅ Assistant management: custom AI assistant configuration and management
- ✅ Agent management: enterprise agent workflow orchestration
- ✅ MCP management: MCP tool integration and management
- ✅ Mini program management: enterprise mini program ecosystem support

**Provider usage**:

- **Built-in providers**: preconfigured mainstream LLM services, **out of the box**
- **No custom providers**: cannot add your own API keys or other providers
- No complex API Key configuration required, ideal for fast onboarding

**Provider limitations**:

- Only system built-in providers and models are available
- Private/self-hosted model services cannot be added
- Your own API keys cannot be used

**Use cases**:

- Small and mid-sized businesses that want fast, out-of-the-box deployment
- Teams without custom provider requirements
- Enterprises that need full features but no provider customization
- Teams with no user count constraints
