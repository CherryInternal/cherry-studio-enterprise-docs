## Edition Overview

Cherry Studio Enterprise comes in two editions — choose the one that fits your needs:

| | **Express** (Cloud-Hosted) | **Premium** (Self-Hosted) |
| --- | --- | --- |
| **Deployment** | No deployment needed — sign up and start | Deploy yourself with Docker / Helm |
| **Data Storage** | Hosted on Cherry Studio cloud | Entirely on your own servers |
| **Ops** | Maintained by Cherry Studio | Maintained by your team |
| **Billing** | Subscription plans (Alipay / bank transfer) | License-based |

<Callout type="info" title="Which edition is right for you?">
- Want **zero deployment, instant access** → [Express Quickstart](/docs/quickstart)
- Need **full data privacy and control** → [Premium Quickstart](/docs/quickstart)
</Callout>

## Data Security and Isolation

Regardless of which edition you choose, Cherry Studio Enterprise provides multiple layers of data protection:

| Security | Express | Premium |
| --- | --- | --- |
| **Data Isolation** | Strict group-based isolation — each organization's data is completely separate and invisible to others | Independent deployment with natural physical isolation |
| **Encryption in Transit** | End-to-end HTTPS encryption | SSL/TLS configured by your team |
| **Authentication** | JWT + OAuth2 with encrypted password storage | JWT with encrypted password storage |
| **Access Control** | Role-based fine-grained permissions (Admin / Group Admin / Member) | Same |
| **API Security** | Independent API keys with per-user access isolation | Same |

<Callout type="info" title="Express Data Isolation">
Express uses a shared infrastructure with application-level isolation. Each organization's data (model configurations, knowledge bases, assistants, files, etc.) is strictly isolated by group. Every API request goes through authentication and permission checks, ensuring access is limited to your organization's data only.
</Callout>
