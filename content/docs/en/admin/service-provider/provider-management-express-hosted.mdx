# 2. Provider Management (Express/Hosted)

### Overview

**Path:** Admin console sidebar > **Model Management** (or Provider Management > Express)

![](/assets/images/zh/admin/provider-management-express-1_zh.webp)

**Cherry Studio Express** is an enterprise-grade, one-stop model aggregation service. Unlike Standard, where you apply for API keys from OpenAI/Anthropic, Express provides a unified quota pool and direct access to top global models (e.g., Claude 3.5, GPT-4o) as well as popular open-source models.

<Callout type="info">
**Core value**:

* **No ops**: No need to manage dozens of API keys and accounts.
* **Network-free**: Use global models directly within enterprise networks without overseas proxies.
* **Unified billing**: All model usage is deducted from the enterprise balance.
</Callout>

***

### 1. Balance Dashboard and Top-up

At the top of the page, you can see enterprise usage:

![](/assets/images/zh/admin/provider-management-express-2_zh.webp)

* **Total balance**: Total funds in the enterprise account.
* **Used**: Amount and percentage consumed in the current period.
* **Available**: Remaining balance available for model calls.
* **Requests**: Total API call count.

**💰 How to top up?** Click **🏦 Corporate payment** in the top right. Supports bank transfer or QR code payments. Balance can be used across all Express models.

***

### 2. Model Categories (Closed vs Open Source)

Express splits models into two categories, switchable via tabs:

![](/assets/images/zh/admin/provider-management-express-3_zh.webp)

#### 🌍 Express Closed Models

Aggregates the strongest closed-source commercial models.

* **Providers**: OpenAI (GPT), Anthropic (Claude), Google (Gemini), X-AI (Grok), etc.
* **Use cases**: Complex reasoning, creative writing, multilingual translation.

#### 🐧 Express Open-Source Models

Aggregates high-cost-performance domestic and open-source models.

* **Providers**: DeepSeek, Alibaba Qwen, Moonshot (Kimi), Z-AI (GLM), etc.
* **Use cases**: Office assistants, Chinese-language tasks, large-scale text analysis at low cost.

***

### 3. Model Publishing and Management

The system provides many models by default. Admins can decide which are visible to employees.

![](/assets/images/zh/admin/provider-management-express-4_zh.webp)

#### Enable/Disable Models

1. Click **𝌡 Manage Models** at the bottom.
2. In the pop-up list, use **+/-** or toggles:
   * **+ (Add)**: Add to available list (visible to employees).
   * **- (Remove)**: Hide from employees.
3. Click **Save**.

#### Manually Add New Models

![](/assets/images/zh/admin/provider-management-express-5_zh.webp)

If a new model (e.g., `gpt-5`) is announced but not yet in the list, you can add it manually:

1. Click **+ Add Model**.
2. Enter the official **Model ID** (e.g., `gpt-5-preview`).
3. Click **Add** to use it immediately.

If you want developers to call these Express models directly from external applications, ask them to use the full `model-id@provider_id` format for the `model` field, and refer to [External App Access (Express Models)](/en/admin/user-management/external-app-access) for the complete setup.

### 5. Billing and Cost Control

**Action:** Click the **⚙️ Settings** icon next to a model.

![](/assets/images/zh/admin/provider-management-express-6_zh.webp)

Although Express uses unified billing, admins can set **internal pricing** to generate department-level cost reports.

* **Input/Output pricing**: You can copy official prices, or set markups/internal prices.
* **Currency**: Switch between USD/CNY. Usage Analytics will calculate costs using this price.

***

### FAQ

**Q: Why do I see "Balance Insufficient" even with balance?**

A: The model might be extremely expensive (e.g., preview models), or your enterprise has a daily spending limit.

**Q: Why did a model suddenly become gray or disappear?**

A: If an upstream provider (e.g., OpenAI) has a major outage, Express will automatically fuse/disable the service to protect stability. It will return after recovery.
