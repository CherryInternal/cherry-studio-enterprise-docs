## Overview

**Path:** Admin console sidebar > **Provider Management**

Cherry Studio Enterprise integrates **Aliyun Bailian** as the core model engine by default. It ships with OpenAI-compatible presets, so enterprises can unlock the full Qwen series and Bailian-hosted open-source models by simply entering an API key.

<Callout type="info">
**Default strategy and extensibility**

* **Default state**: Only the "Aliyun Bailian" provider is shown initially.
* **Extensibility**: Admins can manually add other providers (e.g., local Ollama) to aggregate multiple channels.
</Callout>

***

## Activate the Default Provider

The list already includes an **"Aliyun Bailian"** card.

<img src="/assets/images/en/admin/provider-management-aliyun-bailian-qwen-1_en.webp" alt="" />
1. Click the **Aliyun Bailian** card, then click the toggle in the upper-right corner to enable this provider.
2. **API Key**: Enter the key from the Aliyun Bailian console (starts with `sk-`).
3. **API Host**: Keep the default value.
   * _Address_: `https://dashscope.aliyuncs.com/compatible-mode/v1`
4. Click **Check Connection** to confirm "API Key valid".
5. Click **Save**.

## Manage the Model List

<img src="/assets/images/en/admin/provider-management-aliyun-bailian-qwen-2_en.webp" alt="" />
Bailian provides a long model list. To avoid overwhelming users, admins should trim it.

1. Click **𝌡 Manage Models** under the Bailian card.
2. **Suggested filtering:**
   * Keep `qwen3-max` and `qwen3.6-plus` (latest commercial models).
   * Keep `qwq-plus` (for reasoning tasks).
   * Keep `qwen3-coder-plus` (R&D focused).
   * Keep `qwen3.5-flash` (lightweight high-frequency tasks).
   * Optionally keep `qwen3-32b` or `qwen3-8b` (open-source alternatives).
3. **Hide redundancy:** Click **-** to hide legacy models (e.g., `qwen-max`, `qwen-plus`), dated versions, or tiny models (`0.5b`, `1.7b`).
4. Click **Save**.

## Add Other Providers (Custom)

Although Bailian is the default, Cherry Studio supports other channels.

**Example scenarios:**

* You want to use native **OpenAI** `gpt-4o`.
* You run **Ollama** on a local server.

**Steps:**

1. Click **+ Add Provider** at the bottom of the list.
2. **Provider type**:
   * **OpenAI** (generic compatibility)
   * **Anthropic** (Claude series)
   * **Ollama** (local models)
3. Fill in **API Key** and **API Host**.
   * _Example local Ollama host_: `http://192.168.1.5:11434/v1`
4. The provider's models will appear alongside Bailian models in the client model list.

## FAQ

**Q: What's the difference between `qwen3-max` and the legacy `qwen-max`?**

A: `qwen3-max` is the latest-generation flagship model (262K context), while `qwen-max` points to the previous generation (Qwen 2.5-Max). It is still available but no longer the best choice. Use `qwen3-max` for new projects.

**Q: What's the difference between `qwq-32b` and `qwq-plus`?**

A: `qwq-32b` is an open-weight reasoning model, while `qwq-plus` is Aliyun's commercial closed-source reasoning model. The latter usually provides better reasoning and safety at higher cost.

**Q: Can I still use `qwen-turbo`?**

A: `qwen-turbo` is no longer updated. Migrate to `qwen3.5-flash`, which offers lower cost and longer context (1M).
