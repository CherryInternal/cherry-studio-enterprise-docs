# 4. API Key Management

### Create and Manage Keys

1. In the user list, click the **Keys** button in the **Actions** column for the target user.
2. View and configure settings in the **Key Settings** panel:

<img src="/assets/images/en/admin/user-management-api-key-settings-modal_en.webp" alt="Key settings modal" />
**Configuration details**

<table><thead><tr><th width="111.69921875">Field</th><th width="369.5078125">Description</th><th>Example/Action</th></tr></thead><tbody><tr><td><strong>Key</strong></td><td>System-generated high-strength random string starting with <code>sk-</code>.</td><td>Click the <strong>copy icon</strong> on the right.</td></tr><tr><td><strong>Description</strong></td><td>Note the purpose of this key for audit.</td><td>Example: <code>Dev Environment CI/CD</code></td></tr><tr><td><strong>Expiration time</strong></td><td>Set the key lifetime. 4. Click **Save** to apply.

### Verification and Troubleshooting

After configuration, the user can call Cherry Studio APIs with this key.

* **How to verify?** Send a standard Chat Completion request with the key. If you receive a streaming response, the setup is successful.
* **Common errors `401 Unauthorized` or `Key Expired`:**
  1. Check whether **Expiration time** has passed.
  2. Check if **Enabled status** is off.
  3. Check whether the user's **Group** has valid model channels configured.

If you want to call Enterprise Express models from third-party clients, IDE plugins, or self-built services, continue to [External App Access (Express Models)](/docs/admin/user-management/external-app-access).

<Callout type="info">
**Note:** An API key represents the user's identity and billing permissions. Remind users to keep it safe and never expose it in public code repositories.
</Callout>
