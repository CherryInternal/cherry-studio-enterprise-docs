# User Management

**User Management** is the command center of the Cherry Studio Enterprise admin console. As a system administrator, you configure and govern the digital identities of enterprise members here.

This module is not just CRUD for accounts, but the foundation of **resource allocation** and **security control**. By associating users with specific **groups**, you can precisely control which AI models (Model Providers) and assistant resources different departments or roles can access, effectively managing compute costs and data permissions.

### Core Capabilities

In this module, you can perform the following tasks:

* **Full lifecycle management**: Create accounts, update information (email, password, phone), and deactivate/delete accounts.
* **Fine-grained permission assignment**: Bind users to specific **Groups** so you can configure policies like "R&D uses Claude, Marketing uses GPT-4".
* **Security blocking**: Toggle **Active/Inactive** status to instantly block login and API access for offboarded or anomalous accounts.
* **API gateway credentials**: Generate dedicated **API Keys** for developers or automation scripts to access model capabilities without the client UI.
* **Multi-dimensional auditing**: Provide a list view with search by username and filters by status or group to quickly locate users.

### Quick Start

In the admin console left navigation, choose **User Management** to enter the main page.

<Callout type="warn">
**Important:** A user's **permission scope** depends entirely on their **group**. Before adding users, plan your org structure and model authorization strategy in **Group Management**.
</Callout>
