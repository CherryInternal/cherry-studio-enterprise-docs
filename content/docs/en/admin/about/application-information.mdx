**Path:** **About** > **Application Information**

This panel provides key runtime metrics and downloads. It is the primary dashboard for system health.

<img src="/assets/images/en/admin/about-us-application-info_en.webp" alt="" />
## Version and Status Monitoring

The top area shows the "health" of the current deployment:

<table><thead><tr><th width="104.90625">Field</th><th width="265.25390625">Description</th><th>Purpose</th></tr></thead><tbody><tr><td><strong>Current Version</strong></td><td>Version number of the current deployment (e.g., <code>0.4.10</code>).</td><td>Confirm compatibility and new feature support.</td></tr><tr><td><strong>Latest Version</strong></td><td>Latest officially released version (e.g., <code>0.4.10</code>).</td><td>Compare with current version to determine if an upgrade is needed.</td></tr><tr><td><strong>Build Version</strong></td><td>Internal build timestamp to the minute (e.g., <code>202603311021</code>).</td><td><strong>Key for troubleshooting</strong>. Provide this ID when reporting bugs.</td></tr><tr><td><strong>Service Status</strong></td><td>Real-time backend connectivity check.</td><td>● Running indicates healthy; red means check Docker logs immediately.</td></tr><tr><td><strong>System Update</strong></td><td>After configuring software updates, check and upgrade to the latest version.</td><td>Click "Configure Software Update" to set up first; then check and install updates online.</td></tr></tbody></table>

<Callout type="warn">
**Note:** If **Service status** is abnormal, the frontend may still load but core backend functions (model calls, file parsing) will fail.
</Callout>

## Resource Download Center

Admins can quickly access tools to ensure client/server version alignment:

* **Client downloads:** Links for Windows and macOS clients.
* **Documentation:** Link to official manuals (this doc site).
* **Docker images:** Link to Docker Hub for server upgrades.

<Callout type="info">
**China Mirror:** Users in mainland China can use the domestic mirror for faster and more stable pulls:
```
docker-registry.cherry-ai.com/public/cherry-studio-enterprise-api:v0.4.10
```
</Callout>
