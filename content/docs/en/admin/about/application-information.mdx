# 1. Application Information

**Path:** **About** > **Application Information**

This panel provides key runtime metrics and downloads. It is the primary dashboard for system health.

<img src="/assets/images/en/admin/about-us-application-info_en.webp" alt="" />
#### Version and Status Monitoring

The top area shows the "health" of the current deployment:

<table><thead><tr><th width="104.90625">Field</th><th width="265.25390625">Description</th><th>Purpose</th></tr></thead><tbody><tr><td><strong>Current version</strong></td><td>Major version number (e.g., <code>0.3.8</code>).</td><td>Confirm compatibility and new feature support.</td></tr><tr><td><strong>Build version</strong></td><td>Internal build timestamp to the minute (e.g., <code>202512230648</code>).</td><td><strong>Key for troubleshooting</strong>. Provide this ID when reporting bugs.</td></tr><tr><td><strong>Service status</strong></td><td>Real-time backend connectivity check.</td><td>● Running indicates healthy; red means check Docker logs immediately.</td></tr><tr><td><strong>System update</strong></td><td>Check for new versions.</td><td>If "Up to date", no action; otherwise pull new Docker images.</td></tr></tbody></table>

<Callout type="warn">
**Note:** If **Service status** is abnormal, the frontend may still load but core backend functions (model calls, file parsing) will fail.
</Callout>

#### Resource Download Center

Admins can quickly access tools to ensure client/server version alignment:

* **Client downloads:** Links for Windows and macOS clients.
* **Documentation:** Link to official manuals (this doc site).
* **Docker images:** Link to Docker Hub for server upgrades.
