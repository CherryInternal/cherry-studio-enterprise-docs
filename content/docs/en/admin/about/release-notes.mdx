# 2. Release Notes

**Path:** **About** > **Release Notes**

This module records Cherry Studio's evolution and helps admins understand changes in each update.

<img src="/assets/images/en/admin/about-us-release-notes_en.webp" alt="" />
### Log Structure

Versions are listed in reverse chronological order. Each card contains:

* **Version:** e.g., `v0.4.10`.
* **Release date:** e.g., `2026/03/30`.
* **Changes:** Detailed feature descriptions.

### Change Type Legend

Updates are color-labeled for quick scanning:

<table><thead><tr><th width="193.39453125">Type</th><th width="73.84375">Color</th><th width="298.45703125">Meaning</th><th>Example</th></tr></thead><tbody><tr><td><strong>New Feature</strong></td><td>Green</td><td>Introduces new capabilities beyond optimization.</td><td><code>Support enabling/disabling user accounts</code></td></tr><tr><td><strong>Fix</strong></td><td>Orange</td><td>Fixes known bugs or outages.</td><td><code>Fix Ollama provider metadata and embedding model routing issues</code></td></tr><tr><td><strong>Improvement</strong></td><td>Blue</td><td>Performance optimization, UI adjustments, or experience upgrades.</td><td><code>Add Authorization Header for Anthropic requests</code></td></tr></tbody></table>

<Callout type="info">
**Best practice:** If you see anomalies (incorrect stats, layout issues), check the latest **Fix** entries to see if it is known and resolved.
</Callout>
