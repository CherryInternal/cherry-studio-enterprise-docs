# 1. 创建与基础配置

在知识库列表页，点击右上角的 **+ 添加知识库** 即可开始创建。此步骤决定了数据如何被"理解"和"存储"。

<img src="/assets/images/zh/admin/knowledge-base-create-basic-config_zh.webp" alt="" />
#### 基础参数

<table><thead><tr><th width="123.7421875">参数名称</th><th width="65.84765625">必填</th><th>说明</th></tr></thead><tbody><tr><td><strong>知识库名称</strong></td><td>是</td><td>建议使用具有业务辨识度的名称，如<code>产品技术文档</code>、<code>员工手册 2024</code>。</td></tr><tr><td><strong>描述</strong></td><td>否</td><td>简要说明库内数据的范围，便于其他管理员协作识别。</td></tr><tr><td><strong>嵌入模型 (Embedding)</strong></td><td><strong>是</strong></td><td><strong>核心参数</strong>。负责将文本转化为计算机可理解的向量数据。建议选择支持多语言且维度较高的模型（如 <code>bge-m3</code> 或 <code>text-embedding-v3</code>）。</td></tr></tbody></table>

<Callout type="warn">
**注意：** 知识库创建后，**嵌入模型 (Embedding)** 通常不可直接变更。因为不同模型的向量空间不兼容，若需更换，通常需要重建知识库并重新索引所有文档。
</Callout>

#### 高级设置 (Advanced Settings)

点击 **高级设置** 展开折叠项，这些参数直接影响检索的颗粒度和准确性。

<table><thead><tr><th width="171.5234375">参数</th><th width="112.96484375">推荐值</th><th>深度释义</th></tr></thead><tbody><tr><td><strong>重排模型 (Rerank)</strong></td><td><em>按需开启</em></td><td><strong>二次精筛</strong>。Embedding 模型负责初筛（找得快），Rerank 模型负责对初筛结果进行深度语义打分（找得准）。<br /><strong>建议：</strong> 对准确率要求极高的场景（如法律、医疗）强烈建议开启。</td></tr><tr><td><strong>分块大小 (Chunk Size)</strong></td><td><code>800</code> - <code>1500</code></td><td>模型一次性读取的文本字符数。块太小可能丢失上下文逻辑；块太大可能包含无关噪声。</td></tr><tr><td><strong>重叠大小 (Overlap)</strong></td><td><code>100</code> - <code>200</code></td><td>相邻分块之间的重复字符数。用于确保句子不会在关键处被切断，保持语义连贯性。<br /><strong>建议：</strong> 设为分块大小的 10%-15%。</td></tr></tbody></table>

<Callout type="info">
**配置建议：**

* **通用文档：** Chunk: 1000 / Overlap: 100
* **代码/技术文档：** Chunk: 1500 / Overlap: 200 (代码逻辑较长)
* **碎片化问答：** Chunk: 500 / Overlap: 50
</Callout>
