# 4. 常见问题排查

在使用小程序嵌入网页时，可能会遇到页面无法加载的情况，请参考以下排查指南。

<Callout type="error">
**错误：拒绝了连接请求 / Refused to connect** 如果您在客户端看到类似报错，通常是因为目标网站设置了安全策略。
</Callout>

* **原因分析：** 目标网站在 HTTP 响应头中设置了 `X-Frame-Options: SAMEORIGIN` 或 `DENY`，明确禁止被其他网页（包括 Cherry Studio 的容器）通过 Iframe 嵌入。
* **解决方案：**
  1. **联系网站方：** 如果这是企业内部系统，请联系运维团队允许 Cherry Studio 的域名或 Electron 容器嵌入。
  2. **使用替代链接：** 部分 AI 服务商提供专门用于嵌入的 Embed URL。
  3. **检查协议：** 再次确认填写的 URL 是否为 `https`。

<Callout type="info">
**最佳实践** 在正式向全员发布前，建议管理员先创建一个测试小程序，自己在客户端验证链接是否可以流畅打开并交互，确认无误后再修改名称并正式启用。
</Callout>
