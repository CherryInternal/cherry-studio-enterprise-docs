# 2. 更新日志

**路径：** **关于我们** > **更新日志**

此模块详细记录了 Cherry Studio 的演进历史，帮助管理员了解每次升级带来的具体变化。

![](/assets/images/zh/admin/about-us-release-notes_zh.png)

#### 日志结构说明

系统按时间倒序排列所有历史版本，每个版本卡片包含以下信息：

* **版本号：** 如 `v0.3.8`。
* **发布日期：** 如 `2025/12/23`。
* **变更详情：** 具体的功能点描述。

为了方便快速查阅，更新条目使用了不同颜色的标签进行分类：

<Callout type="info">
**最佳实践：** 在遇到系统异常（如数据统计不准、页面显示错位）时，建议先查看最近一次更新日志的 **修复** 类目，确认是否为已知问题并已在最新版中修复。
</Callout>
