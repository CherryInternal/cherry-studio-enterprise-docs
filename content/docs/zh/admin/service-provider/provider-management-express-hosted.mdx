## 概述

**路径：** 管理后台侧边栏 > **模型管理** (或服务商管理 > Express)

<img src="/assets/images/zh/admin/provider-management-express-1_zh.webp" alt="" />
**Cherry Studio Express** 是企业版专属的一站式模型聚合服务。与旗舰版（Premium）需要您自行去 OpenAI、Anthropic 官网申请 API Key 不同，Express 版通过统一的额度池，直接为您打通全球主流顶尖模型（如 Claude Opus 4.6、Claude Sonnet 4.6、Gemini-3.1、GPT-5.4）以及热门开源模型。

<Callout type="info">
**核心价值**：

* **免运维**：无需管理几十个不同的 API Key 和账号。
* **网络无忧**：无需配置复杂的海外代理，直接在企业内网使用全球模型。
* **统一结算**：所有模型消耗统一从企业余额扣除。
</Callout>

***

## 1. 资金看板与充值

在页面顶部，您可以看到当前企业的资源消耗情况。

<img src="/assets/images/zh/admin/provider-management-express-2_zh.webp" alt="" />
* **总余额**：企业账户当前的资金总量。
* **已使用**：本周期内消耗的金额及占比。
* **剩余可用**：当前可用于调用模型的余额。
* **请求次数**：API 调用的总次数统计。

**💰 如何充值？** 点击右上角的 **💳 余额充值** 按钮，支持企业通过银行转账进行余额充值。充值后，资金可用于所有 Express 渠道的模型调用。

## 2. 模型分类 (闭源 vs 开源)

Express 服务将模型资源划分为两大类，通过顶部的标签页切换：

<img src="/assets/images/zh/admin/provider-management-express-3_zh.webp" alt="" />
### Express 闭源模型

聚合了全球最强大的闭源商业模型。

* **包含厂商**：OpenAI (GPT 系列), Anthropic (Claude 系列), Google (Gemini 系列), X-AI (Grok) 等。
* **适用场景**：处理极其复杂的逻辑推理、创意写作、多语言翻译。

### Express 开源模型

聚合了高性价比的国内及开源大模型。

* **包含厂商**：DeepSeek (深度求索), Alibaba (通义千问 Qwen), Moonshot (Kimi), Z-AI (智谱 GLM) 等。
* **适用场景**：日常办公助手、中文语境处理、大批量文本分析（成本极低）。

## 3. 模型上架与管理

系统默认提供了海量模型，管理员可以决定哪些模型对员工可见。

<img src="/assets/images/zh/admin/provider-management-express-4_zh.webp" alt="" />
### 启用/禁用模型

1. 点击列表底部的 **𝌡 管理模型** 按钮。
2. 在弹出的列表中，通过右侧的 **+/-** 号或开关：
   * **+ (添加)**：将模型添加到可用列表，员工在前端可见。
   * **- (移除)**：隐藏该模型，员工不可见。
3. 点击 **保存**。

### 手动添加新模型

<img src="/assets/images/zh/admin/provider-management-express-5_zh.webp" alt="" />
如果 Express 官方发布了新模型（如 `gpt-5`），但列表尚未自动更新，您可以手动添加：

1. 点击 **+ 添加模型**。
2. 输入官方通告的 **模型 ID** (如 `gpt-5-preview`)。
3. 点击 **确认** 即可立即使用。

如果您希望让开发者在外部应用中直接调用这些 Express 模型，请让其使用完整格式 `模型 ID@provider_id` 作为 `model`，并参考 [外部应用接入（Express 模型）](/zh/docs/admin/user-management/external-app-access) 完成配置。

在 Express 列表中，每个模型名称后面都跟着一排彩色的小图标。这些图标代表了模型的"硬实力"，对于分发模型非常重要。

## 5. 检查连接

在使用模型前，您可以验证与服务商的连通性是否正常。

<img src="/assets/images/zh/admin/provider-management-express-6_zh.webp" alt="" />

1. 点击底部的 **检查连接** 按钮。
2. 在弹窗中选择需要检查的模型。
3. 点击 **确认**，系统将自动测试该模型的可用性。
4. 成功反馈：右下角弹出提示 **"检查成功 API Key 有效"**。

***

## 6. 编辑与删除模型

管理员可以对已上架的模型进行编辑或删除操作。

<img src="/assets/images/zh/admin/provider-management-express-7_zh.webp" alt="" />

### 编辑模型

1. 点击模型右侧的 **⚙️ 设置** 按钮，打开「编辑模型设置」弹窗。
2. 可修改以下信息：
   * **模型名称**：自定义显示名称。
   * **模型描述**：为模型添加说明文字。
   * **能力设置**：开启 **手动指定模型能力** 开关后，可勾选该模型支持的能力标签（推理、视觉、工具调用、默认模型、嵌入、重排序、免费等）。
3. 点击 **保存**。

<Callout type="warn">
请谨慎设置能力标签，错误配置可能导致模型无法正常使用。
</Callout>

### 删除模型

点击模型右侧的 **🗑️ 删除** 按钮，即可将该模型从列表中移除。

## 常见问题 (FAQ)

**Q: 为什么我有余额，但提示 "Balance Insufficient"？**&#x20;

A: 请检查该模型是否属于极其昂贵的特殊模型（如某些预览版），或者您的企业设置了单日消费上限。

**Q: 为什么某个模型突然变灰或消失了？**&#x20;

A: 上游服务商（如 OpenAI）如果出现大面积故障，Express 会自动熔断该服务以保护系统稳定性。待上游恢复后会自动重新上线。
