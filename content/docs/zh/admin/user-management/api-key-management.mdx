# 4. API 密钥管理

#### 创建与管理密钥

1. 在用户列表中，点击目标用户右侧的 **⋮** 图标。
2. 选择 **密钥** (Key 图标)。

![密钥设置弹窗](/assets/images/zh/admin/user-management-api-key-settings-modal_zh.png)

3. 在弹出的 **密钥设置** 面板中进行配置：

![API Key 及过期时间设置](/assets/images/zh/admin/user-management-api-key-expiration-settings_zh.png)

**配置项说明**

<table><thead><tr><th width="111.69921875">参数</th><th width="369.5078125">描述</th><th>示例/操作</th></tr></thead><tbody><tr><td><strong>密钥 (Key)</strong></td><td>系统自动生成的 <code>sk-</code> 开头的高强度随机字符串。</td><td>点击右侧 <strong>复制图标</strong> 一键复制。</td></tr><tr><td><strong>描述</strong></td><td>备注该 Key 的用途，便于审计。</td><td>例：<code>Dev Environment CI/CD</code></td></tr><tr><td><strong>过期时间</strong></td><td>设定 Key 的生命周期。过期后网关将拒绝请求。</td><td>选择具体的日期和时间。</td></tr><tr><td><strong>启用状态</strong></td><td>控制 Key 的即时生效与冻结。</td><td>开启/关闭开关。</td></tr></tbody></table>

4. 点击 **保存** 生效。

#### 验证与排查

配置完成后，用户可使用该 Key 访问 Cherry Studio 的 API 接口。

* **如何验证成功？** 使用该 Key 发起一个标准的 Chat Completion 请求，如果能正常接收到流式响应，说明配置成功。
* **常见错误 `401 Unauthorized` 或 `Key Expired`：**
  1. 检查 **过期时间** 是否已过。
  2. 检查 **启用状态** 是否为关闭。
  3. 检查该所属的 **分组** 是否配置了有效的模型渠道。

<Callout type="info">
**Note:** API Key 等同于用户的数字身份，拥有调用模型的计费权限。请务必提醒用户妥善保管，严禁在公开代码仓库中泄露。
</Callout>
