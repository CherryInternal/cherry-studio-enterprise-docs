## 创建与管理密钥

1. 在用户列表中，点击目标用户右侧 **操作** 列的 **密钥** 按钮。
2. 在弹出的 **密钥设置** 面板中查看和配置：

## 验证与排查

配置完成后，用户可使用该 Key 访问 Cherry Studio 的 API 接口。

* **如何验证成功？** 使用该 Key 发起一个标准的 Chat Completion 请求，如果能正常接收到流式响应，说明配置成功。
* **常见错误 `401 Unauthorized` 或 `Key Expired`：**
  1. 检查 **过期时间** 是否已过。
  2. 检查 **启用状态** 是否为关闭。
  3. 检查该所属的 **分组** 是否配置了有效的模型渠道。

如需在第三方客户端、IDE 插件或自建服务中直接调用企业速通版模型，请继续阅读 [外部应用接入（Express 模型）](/zh/docs/admin/user-management/external-app-access)。

<Callout type="info">
**Note:** API Key 等同于用户的数字身份，拥有调用模型的计费权限。请务必提醒用户妥善保管，严禁在公开代码仓库中泄露。
</Callout>
