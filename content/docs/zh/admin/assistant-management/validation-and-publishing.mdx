# 4. 验证与发布

在将助手分发给全员之前，请务必执行以下验证流程：

1. **功能测试：** 在管理员视图下，打开对话框，选择该助手。
2. **RAG 验证：** 询问一个仅在知识库中存在的问题，确认助手是否能引用文档内容。
3. **工具验证：** 如果启用了 MCP（如联网），尝试发送 `请帮我搜索 Cherry Studio 的最新版本`，确认是否触发工具调用。
4. **授权分发：** 确认无误后，前往 **\[分组管理]** 页面，将该助手授权给目标用户组。

<Callout type="info">
**最佳实践：** 建议先创建一个 "Beta 测试组"，将新配置的复杂助手（特别是带有 MCP 功能的）先分发给该组进行灰度测试，稳定后再全员推广。
</Callout>
