创建完成后，点击 **编辑** 进入详细配置界面。左侧导航栏将其划分为四个核心维度。

### 3.1 提示词设置 (Prompt Settings)

此处管理助手的"人设"与权限。

<img src="/assets/images/zh/admin/assistant-management-advanced-config-1_zh.webp" alt="" />
* **提示词编辑器：** 支持 Markdown 格式，可随时优化系统指令。
*   **提示词可见性：**

    * **开启 (On)：** 允许终端用户在对话详情中查看该助手的系统提示词。
    * **关闭 (Off)：** 仅管理员可见。适用于保护企业核心 Prompt 资产不外泄。

    <img src="/assets/images/zh/admin/assistant-management-advanced-config-2_zh.webp" alt="" />

### 3.3 知识库设置 (Knowledge Base)

为助手挂载"外脑"。在此列表中，您可以勾选预先在 \[知识库管理] 模块中索引好的数据集。

* **运作机制：** 启用后，助手在回答用户问题前，会先在选中的知识库中进行检索，将相关内容作为上下文提供给模型。

### 3.4 MCP 服务器 (MCP Servers)

此模块是助手的"工具箱"。通过启用 MCP (Model Context Protocol) 服务器，您可以赋予助手执行实际操作的能力（如联网搜索、生成图表、读写文件等），使其从单纯的"对话者"进化为"执行者"。

<img src="/assets/images/zh/admin/assistant-management-advanced-config-6_zh.webp" alt="" />
在此页面中，系统会列出 **全局配置** 中已添加的所有可用 MCP 服务。管理员只需通过开关即可完成能力挂载。

**工具示例：**
* **Playwright：** 允许助手访问互联网，读取网页内容。
* **Filesystem：** 允许助手读取或写入服务器上的特定文件目录。

| 字段                   | 说明                                             |
| -------------------- | ---------------------------------------------- |
| **名称 (Name)**        | MCP 服务的唯一标识符（例如 `fetch` 或 `mcp-server-chart`）。 |
| **类型 (Type)**        | 服务的连接类型（通常为 `stdio` 或 `sse`）。                  |
| **描述 (Description)** | 该服务的功能简介。                                      |
| **启用 (Enable)**      | **\[核心操作]** 切换开关。开启后，该助手在对话中即可自动调用此工具。         |

**典型应用场景示例**

* **联网能力 (`fetch` / `search`):** 开启后，助手可以实时访问网页链接，总结新闻或抓取数据。
* **数据可视化 (`mcp-server-chart`):** 允许助手接收用户的数据（如 CSV），并直接绘制出柱状图、折线图等可视化图表。
* **文件操作 (`filesystem`):** 允许助手读取服务器上的特定日志文件或代码库进行分析。

<Callout type="info">
**前置条件：** 此列表中的服务必须先在 **MCP 服务器 (全局)** 模块中完成安装和配置。如果列表为空，请先联系系统管理员添加相应的 MCP 服务。
</Callout>

<Callout type="info">
**配置建议：** 虽然 Cherry Studio 支持为一个助手开启多个 MCP 服务，但建议遵循 **"最小够用原则"**。

* 例如：给"数据分析助手"仅开启 `mcp-server-chart` 和 `filesystem`。
* 原因：过多的工具可能会分散模型的注意力，导致工具调用失败或产生幻觉。
</Callout>
