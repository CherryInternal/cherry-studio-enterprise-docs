## 关于"默认分组" (Default Group)

系统初始化时会自动存在一个 **默认分组**。

* **自动归属机制：** 所有通过 SSO、邀请链接注册或批量导入的新用户，若管理员未在导入时指定分组，将自动归属于默认分组。
*   **最佳实践：**

    <div data-gb-custom-block data-tag="hint" data-style="info" class="hint hint-info"><p>建议将默认分组的权限设置为 <strong>"最小可用集"</strong>（例如：仅允许使用免费模型或低成本模型，仅开放基础助手）。这可以防止新用户在未被管理员分配正式部门前，误操作产生高额的 API 费用。</p></div>

## 故障排查

**Q: 我在分组里勾选了模型，但在客户端里看不到？**

A: 请按以下步骤排查：

1. **全局开关检查：** 前往 **模型管理**，确认该模型本身的状态是 **启用 (Enabled)** 的。如果全局禁用了该模型，即使在分组中授权，用户也无法使用。
2. **同步延迟：** 确认用户的客户端是否已同步配置。建议用户点击客户端左下角的 **设置 > 检查更新**，或重启客户端。

**Q: 用户可以属于多个分组吗？**

A: **不可以**。 在 Cherry Studio 的权限体系中，一个用户同一时间只能归属于一个分组。如需叠加权限（例如某人既属于"研发部"又属于"管理层"），建议创建一个包含综合权限的新分组（如"研发总监组"）。

**Q: 为什么搜索不到特定的模型？**

A: 分组管理中的模型列表数据来源于 **模型管理** 模块。 如果列表中缺失某个模型，说明该模型尚未在系统中注册。请先前往 **模型管理** > **添加模型**，确保您已正确添加并保存了该 Model ID。
