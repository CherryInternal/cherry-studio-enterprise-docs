# 分组管理

### 为什么需要分组管理？

<Callout type="info">
**核心价值：** 分组管理不仅是人员归类工具，更是 **成本控制** 和 **数据安全** 的防火墙。
</Callout>

* **🛡️ 数据与权限隔离** 确保敏感业务部门（如财务、法务）拥有独立的环境，且仅能访问特定的 **Assistant（助手）**，防止敏感信息通过非授权的助手跨部门流转。
* **💰 精细化成本控制** 限制高成本模型（如 GPT-5, Claude Opus 4.6）仅对核心研发或高优先级业务分组开放；普通分组仅授权使用高性价比模型（如 gpt-5.1-mini 或 DeepSeek），从而大幅降低 Token 消耗。
* **🚀 提升协作效率** 为不同职能团队预设专属的助手列表。新员工入职加入特定分组后，即可直接获得工作所需的全部 AI 工具，无需手动逐一配置。

### 界面概览

进入企业管理后台，点击左侧导航栏的 **分组管理**，您将看到分组列表页面。

<img src="/assets/images/zh/admin/group-management-list-page_zh.webp" alt="分组管理列表页" />
列表主要包含以下信息：

* **分组名称**：业务标识（如：研发中心）。
* **描述**：分组用途备注。
* **人数**：当前归属于该组的用户数量。
* **操作**：包含 **设置** 与 **删除** 功能。
