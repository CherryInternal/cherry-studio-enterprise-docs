在页面的顶部，管理员可以通过精细化的筛选条件来定义数据分析的范围。所有下方的图表与列表将随筛选条件的改变而实时联动。

<img src="/assets/images/zh/admin/usage-analytics-filters-controls_zh.webp" alt="" />
## 功能说明

| UI 元素    | 说明                      | 业务场景                    |
| -------- | ----------------------- | ----------------------- |
| **全部分组** | 按企业组织架构（如：产品部、研发部）筛选数据  | 评估特定部门的 AI 资源投入产出比      |
| **全部用户** | 支持搜索并选择特定用户             | 审计个别员工的异常高频调用或 Token 消耗 |
| **时间范围** | 快捷选择（今日、近7天、近30天）或自定义日期 | 对比项目上线前后的用量变化趋势         |
| **刷新**   | 重新加载当前数据                | 获取最新的实时监控数据             |

<Callout type="info">
**注意：** 数据统计可能存在 1-2 分钟的延迟，这是由于分布式系统的日志汇聚处理机制导致的，属于正常现象。
</Callout>
