# 6. 调用日志

这是最底层的审计数据，记录了每一次 API 调用的完整指纹。

<img src="/assets/images/zh/admin/usage-analytics-call-logs-1_zh.webp" alt="" />
### 日志详情

点击列表右侧的 **查看 (眼睛图标)**，将弹出详细信息卡片：

<img src="/assets/images/zh/admin/usage-analytics-call-logs-2_zh.webp" alt="" style={{maxWidth: '50%'}} />
1. **用户：** 发起调用的用户名称。
2. **时间：** 调用发起时间（精确到秒）。
3. **模型：** 使用的模型 ID（如 `google/gemini-3-pro-preview`）。
4. **提供商：** 对应的服务商渠道（如 `Express 闭源模型`）。
5. **Token 使用统计：**
   * **输入 Token:** 本次请求的输入消耗量。
   * **输出 Token:** 本次请求的输出消耗量。
   * **总计 Token:** 输入与输出的合计值。

### 应用场景

* **故障排查：** 当用户反馈"回复截断"或"报错"时，管理员可根据时间戳在此找到对应日志，查看是否因 Token 超限导致。
* **合规审计：** 确认特定时间段内的调用是否符合企业规范。
