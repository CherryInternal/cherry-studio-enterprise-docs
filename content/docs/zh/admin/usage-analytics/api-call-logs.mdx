# 6. 调用日志

这是最底层的审计数据，记录了每一次 API 调用的完整指纹。

![](/assets/images/zh/admin/usage-analytics-call-logs-1_zh.webp)

#### 日志详情

点击列表右侧的 **查看 (眼睛图标)**，将弹出详细信息卡片：

![](/assets/images/zh/admin/usage-analytics-call-logs-2_zh.webp)

1. **用户概况：** 发起人及发起时间（精确到秒）。
2. **模型信息：** 使用的模型 ID（如 `google/gemini-pro`）及对应的服务商渠道。
3. **Token 审计：**
   * **输入 Token:** 消耗量。
   * **输出 Token:** 消耗量。
   * **总计 Token:** 本次调用的计费总额。

#### 应用场景

* **故障排查：** 当用户反馈“回复截断”或“报错”时，管理员可根据时间戳在此找到对应日志，查看是否因 Token 超限导致。
* **合规审计：** 确认特定时间段内的调用是否符合企业规范。
